# 八字穿衣指南 APP 产品需求文档（PRD）

## 版本信息

- 版本：V1.2（最终完整版，含两大核心调整）

- 产品形态：移动端 APP（适配iOS/Android）

- 设计风格：苹果极简风格（清爽、轻量化、无冗余）

- 数据方案：本地存储（不云端同步，卸载即清空）

- 商业模式：IAA广告变现（不干扰核心功能）

- 核心调整：1. 查询页默认选中本人信息；2. 编辑页默认回填已存储信息

## 1. 产品定位

本APP是一款以传统八字命理为核心的轻量化工具类产品，依据用户个人出生八字与当日/当周干支八字，结合五行相生相克规则，输出个性化穿衣色系建议（分为适合、不适合、平三个等级），帮助用户通过穿衣配色规避运势冲突，为日常穿搭提供简单、可执行的运势参考，无需专业命理知识即可轻松使用。

## 2. 目标用户

覆盖所有希望通过穿衣颜色规避运势冲突、追求日常穿搭运势参考的人群，核心场景包括：自用查询、为家人/伴侣/朋友查询，不分年龄层，重点面向对传统命理文化有基础兴趣、偏好轻量化操作的用户。

### 2.1 用户核心痛点

- 缺乏专业命理知识，无法自主解读八字与穿衣色系的关联；

- 每日穿衣选择盲目，担心不适配色影响当日运势；

- 多人查询繁琐，缺乏便捷的命主信息存储与切换功能；

- 编辑命主信息时需重复输入，操作繁琐。

## 3. MVP 核心功能

### 3.1 穿衣色系查询功能

- 八字解析：根据用户输入/选择的命主出生年月日时，自动解析生成命主阴阳八字；

- 流年八字获取：根据用户选择的查询周期（当日/当周），自动匹配对应当日/当周的干支八字；

- 色系建议生成：结合五行相生（木生火、火生土、土生金、金生水、水生木）、相克（木克土、土克水、水克火、火克金、金克木）理论，将命主八字与流年八字匹配，输出五类色系（白色系、黑色/蓝色系、绿色系、红色系、黄色系）的适配建议，分为适合、不适合、平三个等级；

- 核心优化：查询页默认优先选中已存储的“本人”信息，支持手动下拉切换其他命主或临时输入信息。

### 3.2 命主信息管理功能

- 多命主存储：支持存储多个命主信息（本人、家人、朋友等），满足多人查询需求；

- 信息操作：支持对命主信息进行新增、编辑、删除操作；

- 信息标记：支持给命主命名、标记“本人”（仅允许一个命主标记为本人）；

- 核心优化：进入编辑页时，默认回填该命主已存储的全部信息（昵称、出生年月日时间、是否本人状态），无需重复输入。

## 4. 页面架构（底部Tab导航，简洁清晰）

- Tab1：查询（默认首页）—— 核心功能页，用于选择命主、查询穿衣建议；

- Tab2：我的—— 命主信息管理页，用于新增、编辑、删除命主信息。

## 5. 页面交互与详细逻辑（含两大核心调整）

### 5.1 查询页（核心优化页）

#### 5.1.1 页面结构

顶部导航栏（标题：八字穿衣指南）→ 命主选择区域 → 查询周期选择区域 → 查询按钮 → 底部IAA广告位（不遮挡核心功能）。

#### 5.1.2 命主选择交互（核心调整1：默认选中本人）

- 默认行为：若已存储“本人”信息，进入查询页后*自动选中本人*，下拉框显示本人昵称（带“本人”Tag）；若无已存储的本人信息，下拉框显示“请选择命主”，提示用户选择或新增。

- 命主切换：点击下拉框，展开已存储的所有命主列表（按昵称排序），点击任意命主即可切换选中状态，下拉框同步显示该命主昵称（本人命主标注“本人”Tag）。

- 临时输入：提供“临时输入”入口（按钮），点击后跳转至临时输入页，填写命主出生年月日时间后返回查询页，查询页自动填充该临时信息（临时信息不存入本地，仅本次查询有效）。

- 校验规则：命主选择为必填项，若未选择已存储命主且未临时输入信息，点击查询按钮时，弹出轻提示“请选择或输入命主信息”。

#### 5.1.3 查询周期选择

- 选项：当日（默认选中）、当周，两者互斥单选；

- 交互：点击对应选项切换选中状态，选中态用品牌主色标注，未选中态为浅灰色，视觉区分清晰。

#### 5.1.4 查询按钮

- 样式：苹果极简风，宽度100%，高度48px，圆角12px，品牌主色背景，白色粗体文字（16px），适配手机端点击尺寸（触控目标≥44px）；

- 交互：点击后校验命主信息，校验通过则跳转至查询结果页；未通过则弹出轻提示，不跳转。

### 5.2 查询结果页

页面结构：顶部导航栏（可返回查询页）→ 命主基础信息模块 → 穿衣色系建议模块 → 底部IAA广告位。

#### 5.2.1 模块1：命主基础信息

卡片式布局（白色背景、圆角12px、浅阴影），内边距16px，展示内容：

- 命主名称：显示选中命主的昵称，若为本人，在昵称右上角标注“本人”Tag（红色背景、白色文字、12px、圆角4px）；

- 出生信息：格式为“出生：XXXX年XX月XX日 XX:XX”（24小时制，精确到分钟）；

- 命主八字：格式为“阴阳八字：XXXXXX”（解析后的完整八字）。

#### 5.2.2 模块2：穿衣色系建议

根据查询周期（当日/当周）自动切换展示形式，均按“适合、不适合、平”三类聚合展示五类色系，视觉区分清晰。

- 当日建议：

- 标题：“今日穿衣建议”（后缀显示日期+星期，如“2026年3月23日 星期日”）；

- 展示形式：分三栏布局，每栏对应一个等级，标题标注对应颜色（适合：绿色#34C759、不适合：红色#FF3B30、平：橙色#FF9500）；

- 内容：每栏内列出对应等级的色系（白色系、黑色/蓝色系、绿色系、红色系、黄色系），若某等级无对应色系，显示“—”（浅灰色14px）。

当周建议：

- 标题：“本周穿衣建议”；

- 展示形式：表格布局（白色背景、圆角12px、浅边框），表头为“日期”“穿衣建议”；

- 内容：每行对应一天，日期格式为“XX月XX日 周X”，穿衣建议同当日逻辑，按三类等级展示五类色系；若某一天无建议，显示“暂无建议”（浅灰色14px，居中）。

### 5.3 我的页面（命主信息管理页）

页面结构：顶部导航栏（标题：我的）→ 新增人员模块 → 已存储命主列表模块 → 底部IAA广告位。

#### 5.3.1 模块1：新增人员

- 元素：“新增人员”按钮（苹果极简风，宽度100%，高度44px，品牌主色背景，白色文字15px，圆角8px）；

- 交互：点击按钮，弹出新增表单页，填写信息后点击确认，完成新增并返回我的页面，列表自动刷新。

#### 5.3.2 模块2：已存储命主列表

- 展示形式：列表布局，每行对应一个命主，高度60px，内边距16px，边框底部1px solid浅灰色（#E5E5E5）；

- 每行元素：命主昵称（15px，深灰色#333333）+ 编辑按钮（图标，浅灰色#666666，点击态变品牌主色）+ 删除按钮（图标，浅灰色#666666，点击态变红#FF3B30）；本人命主在昵称右上角标注“本人”Tag；

- 空状态：若暂无命主信息，显示“暂无命主信息，点击新增人员添加”（14px，浅灰色#999999，居中，顶部间距50px）；

- 交互：

- 点击编辑按钮：跳转至编辑表单页，默认回填该命主已存储的全部信息（核心调整2）；

- 点击删除按钮：弹出确认弹窗（白色背景、圆角12px，提示文字“确定删除该命主信息吗？”），弹窗底部有“取消”（浅灰色按钮）和“确认删除”（红色按钮），点击确认后删除该命主信息，列表自动刷新；点击取消，弹窗关闭，不执行删除操作。

### 5.4 新增页 & 编辑页（表单页，核心调整2：编辑页回填信息）

两个页面表单结构完全一致，仅初始填充内容不同，均为弹窗式布局（白色背景、圆角12px、内边距16px，适配手机端尺寸），包含表单字段和确认按钮，所有字段均为必填项。

#### 5.4.1 统一表单字段

- 昵称输入框：单行输入，placeholder为“请输入命主昵称”，文字15px，深灰色#333333，输入框边框1px solid浅灰色#E5E5E5，圆角8px，高度44px；

- 出生日期选择器：弹出日期选择面板，精确到日，默认不选中（新增页）/ 回填已存储日期（编辑页），选择后显示格式为“XXXX年XX月XX日”；

- 出生时间选择器：弹出时间选择面板（24小时制），精确到分钟，默认不选中（新增页）/ 回填已存储时间（编辑页），选择后显示格式为“XX:XX”；

- 是否本人：开关组件，默认关闭（新增页）/ 回填已存储状态（编辑页，原本是本人则开关打开，否则关闭），开关开启时背景为品牌主色#007AFF，关闭时为浅灰色#E5E5E5。

#### 5.4.2 新增页逻辑（无回填）

- 所有表单项初始为空，“是否本人”开关默认关闭；

- 点击“确认保存”按钮：校验所有必填项不为空，校验通过则新增一条命主信息并本地存储，返回我的页面并刷新列表；校验未通过（某字段为空），弹出轻提示“请填写完整信息”，不关闭表单页。

#### 5.4.3 编辑页逻辑（核心调整2：默认回填）

- 进入编辑页时，系统自动读取该命主的本地存储信息，完成所有表单项的回填：

- 昵称输入框：显示该命主已保存的昵称，若昵称为空，显示placeholder“请输入命主昵称”；

- 出生日期选择器：默认选中并显示该命主已存储的出生年月日；

- 出生时间选择器：默认选中并显示该命主已存储的出生时间（精确到分钟）；

- 是否本人开关：默认状态与该命主当前的存储状态一致（原本是本人则打开，否则关闭）。

点击“确认保存”按钮：校验所有必填项不为空，校验通过则用当前表单内容覆盖更新该命主的本地存储信息，返回我的页面并刷新列表；校验未通过，弹出轻提示“请填写完整信息”，不关闭表单页；

特殊规则：若编辑时将某命主标记为“本人”，则自动取消原有“本人”命主的标记（确保仅一个命主为本人），查询页默认选中更新后的本人信息。

## 6. 典型用户操作流程

### 6.1 首次使用流程

打开APP → 进入查询页（无本人信息，显示“请选择命主”）→ 切换至我的页面 → 点击“新增人员”→ 填写昵称、出生年月日时间，开启“是否本人”开关 → 点击“确认保存”→ 返回我的页面（显示本人信息）→ 切换至查询页（自动选中本人）→ 选择查询周期（默认当日）→ 点击“立即查询”→ 查看穿衣建议结果。

### 6.2 日常查询流程

打开APP → 进入查询页（自动选中本人）→ 确认查询周期（或切换为当周）→ 点击“立即查询”→ 查看结果；如需查询家人信息，点击命主下拉框，切换至对应家人命主 → 点击查询 → 查看结果。

### 6.3 命主编辑/删除流程

打开APP → 切换至我的页面 → 找到目标命主 → 点击“编辑”→ 编辑表单自动回填原有信息 → 修改所需内容（含出生时间）→ 点击“确认保存”→ 列表刷新；或点击“删除”→ 确认弹窗点击“确认删除”→ 命主信息删除，列表刷新。

|---|---|---|
|18px|导航栏标题、页面主标题|粗体|

### 7.3 交互规范

- 按钮尺寸：主按钮最小高度48px，次级按钮最小高度40px，所有可点击元素触控目标≥44px，适配手机端点击习惯；

- 圆角规范：卡片、表单、按钮圆角统一为12px（次级按钮8px），视觉统一；

- 间距规范：页面内元素间距统一为16px，列表项间距16px，保证视觉清爽、不拥挤；

- 提示规范：所有校验提示、操作反馈均为轻提示（居中显示，3秒后自动消失），不遮挡核心内容；

- 适配规范：所有页面、元素均适配iOS/Android主流手机尺寸，禁止出现内容截断、按钮无法点击的情况。

## 8. 数据存储方案

### 8.1 存储范围

所有数据均采用本地存储，不进行云端同步，用户卸载APP后，所有存储信息自动清空，保障用户隐私。

### 8.2 存储字段（命主信息）

|字段名称|字段类型|是否必填|备注|
|---|---|---|---|
|命主ID|字符串|是|唯一标识，自动生成|
|昵称|字符串|是|用户自定义，用于展示和选择|
|出生年月日|日期格式|是|格式：XXXX-XX-XX|
|出生时间|时间格式|是|格式：HH:MM（24小时制，精确到分钟）|
|是否本人|布尔值|是|true=本人，false=非本人，仅允许一个true|
|命主八字|字符串|是|根据出生年月日时间解析生成，自动存储|
### 8.3 存储逻辑

- 新增命主：自动生成唯一命主ID，解析八字后，将所有字段存入本地；

- 编辑命主：根据命主ID，用新表单内容覆盖原有字段，若修改“是否本人”状态，同步更新其他命主的该字段（确保唯一）；

- 删除命主：根据命主ID，删除该命主的所有存储字段；

- 查询页默认选中：读取本地存储中“是否本人”为true的命主，若存在则自动选中，若不存在则显示默认提示。

## 9. 商业模式（IAA广告变现）

### 9.1 广告展示位置

- 查询页底部：横幅广告，宽度100%，高度适配广告内容，不遮挡查询按钮和核心操作区域；

- 查询结果页底部：横幅广告，与结果模块间距16px，不遮挡命主信息和穿衣建议；

- 我的页面底部：横幅广告，与命主列表间距16px，不遮挡新增按钮和列表内容。

### 9.2 广告规范

- 广告风格：贴合苹果极简风，避免色彩过于艳丽、内容杂乱的广告，降低用户反感度；

- 广告类型：仅接入原生广告、横幅广告，不使用插屏广告、弹窗广告，不干扰用户核心操作；

- 变现节奏：MVP版本上线后，仅接入查询结果页底部广告，保障核心用户体验；用户量达到10万+日活后，新增查询页、我的页面底部广告，提升变现效率。

## 10. 核心需求调整总结
