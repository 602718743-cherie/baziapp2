# 八字穿衣指南 - 微信小程序

## 项目简介

这是一款基于中国传统八字命理的微信小程序，帮助用户根据个人生辰八字和当日五行属性，获得穿衣配色建议。

## 功能特性

- ✅ **八字解析**: 根据出生年月日时自动解析生成八字
- ✅ **运势查询**: 支持查询今日和本周的穿衣运势
- ✅ **人员管理**: 支持存储多个人员信息，快速切换查询
- ✅ **本地存储**: 所有数据本地存储，保护隐私
- ✅ **精美UI**: 苹果极简风格，简洁美观

## 快速开始

### 1. 环境准备

- 安装 [微信开发者工具](https://developers.weixin.qq.com/miniprogram/dev/devtools/download.html)
- 注册微信小程序账号（或使用测试号）

### 2. 导入项目

1. 打开微信开发者工具
2. 点击"导入项目"
3. 选择 `miniprogram` 目录
4. 填写 AppID（可使用测试号）
5. 点击"导入"

### 3. 运行项目

项目导入后会自动编译，可以在开发者工具中预览：

- **模拟器**: 查看页面效果
- **真机调试**: 扫码在手机上调试
- **编译**: 点击"编译"按钮刷新

### 4. 体验流程

1. **首次使用**:
   - 切换到"我的"页面
   - 点击"新增人员"
   - 填写姓名、出生日期、出生时间
   - 开启"是否本人"开关
   - 点击"确认"保存

2. **查询运势**:
   - 回到"查询"页面
   - 系统自动选中本人
   - 选择"今日"或"本周"
   - 点击"立即查询"
   - 查看穿衣建议结果

3. **管理人员**:
   - 在"我的"页面可以编辑或删除人员
   - 支持存储多个家人朋友信息

## 核心功能说明

### 1. 八字计算

基于出生年月日时计算：
- **年柱**: 根据公历年份计算天干地支
- **月柱**: 根据年份和月份计算
- **日柱**: 根据具体日期计算
- **时柱**: 根据出生时辰计算

### 2. 五行生克

- **五行**: 木、火、土、金、水
- **相生**: 木生火、火生土、土生金、金生水、水生木
- **相克**: 木克土、土克水、水克火、火克金、金克木

### 3. 穿衣建议

根据命主五行与当日五行的生克关系，给出三级建议：
- **适合**: 状态顺畅时推荐的颜色
- **不适合**: 应该避免的颜色
- **平**: 影响不大的颜色

### 4. 颜色映射

- 木 → 绿色系
- 火 → 红色系
- 土 → 黄色/棕色系
- 金 → 白色系
- 水 → 黑色/蓝色系

## 技术栈

- **框架**: 微信小程序原生框架
- **语言**: JavaScript (ES6+)
- **样式**: WXSS (类CSS)
- **存储**: 微信小程序 Storage API
- **UI风格**: 苹果极简风格

## 数据存储

- **存储方式**: wx.setStorageSync / wx.getStorageSync
- **存储内容**: 人员信息（姓名、生辰、是否本人）
- **数据持久化**: 本地存储，卸载清空
- **隐私保护**: 所有数据仅存储在本地，不上传服务器

## 页面功能详解

- **功能**: 运势查询主页
- **组件**:

1. **AppID配置**: 
   - 测试时可使用"测试号"
   - 正式发布需注册小程序账号
   - 在 `project.config.json` 中修改 `appid`

2. **API接口**:
   - 当前为纯前端计算，不依赖后端

3. **图标资源**:
   - TabBar图标需自行准备（或使用文字）
   - 建议尺寸：81px * 81px

4. **兼容性**:
   - 基础库版本：2.33.0+
   - 支持iOS和Android

5. **隐私政策**:
   - 发布前需配置隐私政策
   - 说明数据存储方式

## 常见问题

### Q: 如何修改AppID？

A: 在 `project.config.json` 中修改 `appid` 字段，或在开发者工具详情中修改。

### Q: TabBar图标不显示？

A: 需要准备图标资源，或在 `app.json` 中移除 `iconPath` 和 `selectedIconPath`，使用纯文字。

### Q: 如何清空本地数据？

A: 在开发者工具中选择"清除缓存" → "清除数据缓存"，或在真机上长按小程序删除。

### Q: 如何接入后端API？

A: 修改 `app.js` 中的 `apiBase`，并在需要的页面使用 `wx.request` 调用接口。
